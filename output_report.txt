This report documents representative test cases for countnames.c and shell1.c after the pipe protocol update. The update added a typed message header to the parent-child communication path while preserving the expected visible output.

Build Verification:
gcc -Wall -Wextra -g -o countnames countnames.c
gcc -Wall -Wextra -g -o shell1 shell1.c

Result:
Both programs compiled successfully with no warnings.

Test Case 1: Direct countnames run on names.txt
Command:
./countnames test/names.txt

Expected behavior:
- Create one PID.out file with the names and counts found in the file.
- Create one PID.err file with warnings for empty lines.

Observed PID.out contents:
Nicky: 1
Dave Joe: 2
Yuan Cheng Chang: 3
John Smith: 1

Observed PID.err contents:

Warning - file test/names.txt line 5 is empty.

Notes:
- This confirms direct file parsing still works.
- Empty lines are skipped and reported in the .err file.

Pipe Protocol Verification:
- The code now sends a MessageHeader plus NameCountData together as one fixed-size packet.
- This avoids mismatching a header from one child with payload data from another child.
- shell1 checks the message type and payload size before merging counts.

The updated implementation still compiles and produces the expected direct and merged outputs. The message-header enhancement only changes the internal communication format and does not change the intended visible behavior of the assignment.
